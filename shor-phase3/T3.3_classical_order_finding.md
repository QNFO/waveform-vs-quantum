# T3.3: Classical ORDER-FINDING Upper Bounds

## The Structural Question

Shor's algorithm reduces FACTORING to ORDER-FINDING with a randomized polynomial-time oracle reduction:

$$
\text{FACTORING} \in \text{BPP}^{\text{ORDER-FINDING}}
$$

This means: if you have a black box for computing $\operatorname{ord}_N(a)$, you can factor $N$ efficiently classically (the post-processing is entirely classical — modular exponentiation and GCD).

But this raises a **subtle structural question**:

> **Is ORDER-FINDING strictly easier than FACTORING? Could ORDER-FINDING be in BPP even if FACTORING is not?**

If ORDER-FINDING $\in$ BPP, then Shor's reduction would still be logically valid, but the *quantum advantage* would disappear — factoring would become classically efficient through the "back door" of period-finding, without needing a quantum computer.

---

## 1. The Two-Direction Relationship

### 1.1 FACTORING $\leq_R$ ORDER-FINDING (Shor's Direction)

This is the well-known randomized reduction:

1. If $N$ is even or a perfect power, factor trivially.
2. Choose $a \in \{2, 3, \ldots, N-2\}$ uniformly at random.
3. Compute $r = \operatorname{ord}_N(a)$. ← **Oracle call**
4. If $r$ is odd, restart.
5. If $a^{r/2} \equiv -1 \pmod{N}$, restart.
6. Output $\gcd(a^{r/2} \pm 1, N)$.

Success probability $\geq 1 - 2^{-(k-1)}$ where $k$ is the number of distinct odd prime factors. Expected oracle calls: $O(1)$.

### 1.2 ORDER-FINDING $\leq_T$ FACTORING (The Reverse Direction)

If we can factor $N$, can we compute $\operatorname{ord}_N(a)$ efficiently? The answer is: **almost, but with a catch**.

Given the factorization $N = \prod p_i^{e_i}$:

1. By CRT, $\operatorname{ord}_N(a) = \operatorname{lcm}(\operatorname{ord}_{p_1^{e_1}}(a), \ldots, \operatorname{ord}_{p_k^{e_k}}(a))$.
2. For each prime power, $\operatorname{ord}_{p_i^{e_i}}(a)$ divides $\phi(p_i^{e_i}) = p_i^{e_i-1}(p_i-1)$.
3. We can factor $\phi(p_i^{e_i})$ by factoring $p_i-1$.
4. Then use the **Pohlig–Hellman algorithm** to compute the order: for each prime $q \mid \phi(p_i^{e_i})$, determine the $q$-adic valuation of the order by testing $a^{\phi/q^j} \equiv 1 \pmod{p_i^{e_i}}$.

**The catch:** Step 3 requires factoring $p_i - 1$. If $p_i$ is a large prime (e.g., 1024-bit RSA prime), then $p_i - 1$ is composite and factoring it might be as hard as factoring $N$ itself.

However, this is about factoring a *different* number ($p_i - 1$) which could have its own structure. In the generic case, $p_i - 1$ is about the same size as $p_i$, so factoring it is comparable in difficulty to factoring $p_i$ itself — which is exactly the original problem.

| Direction | Complexity | Notes |
|-----------|------------|-------|
| FACTORING → ORDER-FINDING | $\text{BPP}^{\text{ORDER-FINDING}}$ | Shor's reduction; $O(1)$ oracle calls |
| ORDER-FINDING → FACTORING | $\text{BPP}^{\text{FACTORING}+\text{FACTORING of }(p_i-1)}$ | Requires factoring each $p_i-1$ recursively |

This establishes approximate **polynomial-time Turing equivalence under generic hardness assumptions**, but with the important nuance that ORDER-FINDING may be strictly easier.

## 2. Classical Algorithms for ORDER-FINDING

### (a) Naive Enumeration

Compute $a^x \bmod N$ for $x = 1, 2, \ldots$ until $a^x \equiv 1$.

**Complexity:** $O(r \cdot M(\log N))$ where $M(n)$ is the cost of $n$-bit multiplication. Since $r \leq \phi(N) < N$, this is:

$$
O(N \operatorname{polylog} N) = \exp(O(\log N)) \quad \text{— EXPONENTIAL}
$$

### (b) Factor $\phi(N)$ First

Compute $\phi(N)$ then factor it to find divisors of the order.

**Problem:** Computing $\phi(N)$ without factoring $N$ is as hard as factoring $N$, since $\phi(N) = (p-1)(q-1)$ for RSA semiprime, and knowing $\phi(N)$ allows immediate recovery of $p, q$ via $p+q = N - \phi(N) + 1$, $p-q = \sqrt{(p+q)^2 - 4N}$.

**Result:** Circular — no advantage over direct factoring.

### (c) Baby-Step Giant-Step (Shanks)

For computing $\operatorname{ord}_N(a)$:

1. Set $m = \lceil \sqrt{\phi(N)} \rceil \leq \lceil \sqrt{N} \rceil$.
2. **Baby steps:** Compute and store $(a^j \bmod N, j)$ for $j = 0, 1, \ldots, m-1$.
3. **Giant steps:** Compute $a^{-m} \bmod N$, then iteratively multiply to find $a^t \equiv a^{-im} = a^{t - im}$.

**Complexity:** $O(\sqrt{r} \operatorname{polylog} N)$ time and $O(\sqrt{r} \log N)$ space.

Since $r$ can be as large as $\lambda(N) \approx N$ (e.g., when $N$ is prime and $a$ is a primitive root):

$$
O(\sqrt{N} \operatorname{polylog} N) = \exp\left(O\left(\frac{\log N}{2}\right)\right) \quad \text{— EXPONENTIAL}
$$

This is better than naive enumeration ($\sqrt{N}$ vs. $N$) but still exponential. For a 2048-bit modulus, $\sqrt{N} \approx 2^{1024}$, which is astronomically large.

### (d) Pollard's Rho for Order Finding

Pollard's rho algorithm can find the order in $O(\sqrt{p})$ where $p$ is the smallest prime factor of the order. In the worst case, this is $O(\sqrt{N})$ again.

**Variant:** Pollard's rho for discrete logarithms can be adapted for order-finding in $O(\sqrt{r})$ expected time. Still exponential in $\log N$.

### (e) Subexponential Algorithms?

The Number Field Sieve (GNFS) achieves:

$$
\exp\left(O\left((\log N)^{1/3} (\log \log N)^{2/3}\right)\right)
$$

for FACTORING. **Is there an analogous subexponential algorithm for ORDER-FINDING?**

**Answer: Not in general.** The subexponential algorithms for factoring exploit the multiplicative structure of integers (smoothness over a factor base). ORDER-FINDING is a problem in a cyclic group — it asks for the order of a specific element. This is structurally similar to the **discrete logarithm problem** (DLP).

In group-theoretic terms:
- **DLP:** Given $g, h \in G$, find $x$ such that $g^x = h$.
- **ORDER-FINDING:** Given $a \in G$, find $r$ such that $a^r = 1$.

ORDER-FINDING reduces to DLP in the subgroup $\langle a \rangle$:
- If we can compute $\log_a(1)$, that gives the order.
- Conversely: $\log_g(h) \bmod r$ where $r = \operatorname{ord}(g)$.

However, the **Index Calculus** method, which gives subexponential algorithms for DLP in $\mathbb{F}_p^\times$, works because the group has a "natural" representation as integers modulo $p$, allowing smoothness testing.

For ORDER-FINDING in $(\mathbb{Z}/N\mathbb{Z})^\times$:
- Index calculus can be applied to **compute the group order** $\phi(N)$ — but that's equivalent to factoring $N$.
- Index calculus can solve DLP in $(\mathbb{Z}/N\mathbb{Z})^\times$ — but only if you know the group structure, which requires factoring $N$.

**Key insight:** The hardness of ORDER-FINDING in $(\mathbb{Z}/N\mathbb{Z})^\times$ is **not** from the group structure being unknown — it's from $N$ being composite and the group order (which is needed for exponent arithmetic) depending on the unknown factorization.

## 3. The Deep Structural Question

### 3.1 Could ORDER-FINDING Be in BPP?

Consider this hypothetical: someone discovers a classical polynomial-time algorithm for ORDER-FINDING that **does not** rely on factoring $\phi(N)$. What would this imply?

- By Shor's reduction, FACTORING $\in$ BPP (since we can call ORDER-FINDING as a subroutine).
- RSA, Diffie-Hellman, and all integer-factoring-based cryptography would be broken.
- But the algorithm wouldn't need to factor $N$ — it would compute orders directly.

**Is this plausible?** We don't know. The question is equivalent to:

> Is ORDER-FINDING complete for some complexity class that includes FACTORING?

Known facts:
- Both FACTORING and ORDER-FINDING are in **NP $\cap$ coNP**.
- Neither is known to be NP-hard (and likely isn't, unless PH collapses).
- ORDER-FINDING is **at least as hard as** FACTORING (since FACTORING $\leq_R$ ORDER-FINDING).
- FACTORING is **at least as hard as** ORDER-FINDING modulo the factoring of $p_i-1$.

So they are approximately in the same complexity class, with neither proven strictly harder than the other.

### 3.2 What If ORDER-FINDING $\in$ BPP?

Three scenarios emerge:

| Scenario | FACTORING | ORDER-FINDING | Outcome |
|----------|-----------|---------------|---------|
| (a) | $\in$ BPP | $\in$ BPP | Both easy. Quantum advantage vanishes. RSA broken classically. |
| (b) | $\notin$ BPP | $\in$ BPP | **Impossible** (by Shor's reduction: FACTORING $\leq_R$ ORDER-FINDING, so BPP closure gives contradiction). |
| (c) | $\in$ BPP | $\notin$ BPP | Logically possible but counterintuitive: we can factor efficiently but can't compute orders. Requires an algorithm that factors $N$ without producing $\phi(N)$ or individual orders. This is actually what GNFS does! GNFS factors $N$ but does not compute $\phi(N)$ or individual element orders. |
| (d) | $\notin$ BPP | $\notin$ BPP | Consensus view. Both hard. Quantum advantage meaningful. |

### 3.3 Scenario (c) Under the Microscope

GNFS demonstrates that factoring $N = p \cdot q$ does **not** require computing $\phi(N)$ or the order of any specific element. The algorithm works by:
1. Finding polynomials over $\mathbb{Z}$ that share a root modulo $N$
2. Sieving for smooth values
3. Solving a large sparse linear system modulo 2

At no point does GNFS compute $\operatorname{ord}_N(a)$ for any $a$. So it's entirely possible that FACTORING is easier than ORDER-FINDING in practice (and GNFS is the evidence: it runs in subexponential time, while the best known ORDER-FINDING algorithms are exponential).

**This is a crucial observation:** The best classical factoring algorithm (GNFS, subexponential) is **faster** than the best classical order-finding algorithm (BSGS or Pollard rho, exponential). If we only had classical computers:

- **Factoring RSA-2048:** GNFS would need $\approx 2^{112}$ operations (estimated, though with astronomical memory requirements; actual estimates suggest it's infeasible but not as hard as $2^{1024}$).
- **Order-finding on RSA-2048:** BSGS would need $\approx 2^{1024}$ operations — far worse.

**This is the asymmetry that Shor exploits.** Shor's quantum subroutine replaces the exponential $\sqrt{N}$ order-finding with a polynomial-time quantum procedure $O(\log^3 N)$, turning the bottleneck into the fast path.

## 4. Formal Complexity Landscape

### 4.1 Known Reductions

```
ORD-FINDING
    ↑ (randomized poly-time: Shor 1994)
FACTORING
    ↓ (Turing: factor N, then factor (p_i-1) for each prime factor)
ORD-FINDING
```

This forms a near-cycle, broken only by the need to recursively factor integers in the reverse direction.

### 4.2 The Exact Relationship

Define:

- $\text{OF}_N(a)$ = problem of computing $\operatorname{ord}_N(a)$
- $\text{FACT}(N)$ = problem of finding a non-trivial factor of composite $N$

Then:

$$
\text{FACT} \in \text{BPP}^{\text{OF}} \quad \text{and} \quad \text{OF} \in \text{BPP}^{\text{FACT}, \text{FACT}^*}
$$

where $\text{FACT}^*$ denotes recursive factoring of the $(p_i - 1)$ components.

In the **generic case** (random integers), $p_i - 1$ is itself composite with large prime factors, and factoring it is comparable in difficulty to factoring $p_i$. This makes the reverse reduction recursive, and it's not clear that this recursion terminates in polynomial time without additional assumptions.

### 4.3 The "Smooth Order" Exception

If $p_i - 1$ is **$B$-smooth** for small $B$, then:
- Factoring $p_i - 1$ is easy (via trial division or Pollard $p-1$).
- Computing $\operatorname{ord}_{p_i}(a)$ via Pohlig–Hellman is efficient.
- Therefore ORDER-FINDING for those specific $N$ is classically easy.

This is exactly the **Pollard $p-1$ attack**: if $p-1$ is smooth for a factor $p$ of $N$, then $N$ can be factored directly. The same smoothness property makes both order-finding AND factoring easy.

**Implication:** The hardness of ORDER-FINDING is tightly coupled to the lack of smoothness in $p_i - 1$, which is exactly the same property that protects against Pollard $p-1$ factoring.

## 5. Summary: The Structural Question Answered

### Is there a classical algorithm for ORDER-FINDING faster than FACTORING?

| Algorithm | Complexity | Faster than GNFS for FACTORING? |
|-----------|------------|:---:|
| Naive enumeration | $\exp(O(\log N))$ | No — exponentially worse |
| Baby-step giant-step | $\exp(O(\log N))$ | No — exponentially worse |
| Pollard rho | $\exp(O(\log N))$ | No — exponentially worse |
| Index calculus in $(\mathbb{Z}/N\mathbb{Z})^\times$ | Requires factoring $\phi(N)$ | Circular |
| Any known subexponential for ORDER-FINDING? | **None known** | — |

**Answer: ORDER-FINDING appears to be classically HARDER than FACTORING in practice**, despite the Shor reduction going the other direction. The best classical factoring algorithm (GNFS) is subexponential, while the best classical order-finding algorithms are exponential.

The reduction $\text{FACTORING} \leq_R \text{ORDER-FINDING}$ only means that if ORDER-FINDING were easy, FACTORING would also be easy. It does **not** mean ORDER-FINDING is easier — and the evidence suggests the opposite.

### The Quantum Advantage Deconstructed

Shor's algorithm works precisely because:

1. ORDER-FINDING is **exponentially hard** classically ($\Omega(\sqrt{N})$).
2. ORDER-FINDING is **polynomially easy** quantumly ($O(\log^3 N)$ via QFT).
3. FACTORING reduces to ORDER-FINDING with constant overhead.

The quantum advantage is therefore:

$$
\boxed{\text{Quantum speedup} = \frac{O(\sqrt{N})}{O(\log^3 N)} \approx \frac{2^{n/2}}{n^3} \quad \text{for } n\text{-bit } N}
$$

This is an **exponential-over-polynomial** separation — one of the strongest known complexity separations between quantum and classical computation for a practically relevant problem.
